# 一键配置 · 扩协议实现规格（照抄级）

> 来源：2026-07-10 多智能体审计工作流（`oneclick-deep-audit`）的扩协议规格专家 agent，基于 golden fixtures + 现有 `oneclick.go` 推导。
> 目的：为【一键配置】新增免证书预设提供可直接照抄的 Go 实现规格。M2 已有 reality / ss2022 两预设，本文补 vmess / vlessTcp。

---

## 硬约束

- **免证书**：目标是裸 IP VPS / 本地即用，不依赖域名与 TLS 证书。
- 权威形状依据：`frontend/src/test/golden/fixtures/inbound-full/` 下 `vmess-tcp-tls.json` / `vless-ws-tls.json` / `vless-tcp-reality.json` / `shadowsocks-tcp-2022.json`。

---

## 预设 1：VMess + TCP（无 TLS，免证书）

- **proto 常量**：`model.VMESS`
- **needs_cert**：false
- **keygen**：none —— 只需 `uuid.New().String()` 生成 client id（与 `realityInboundParts` 里 `clients[0].id` 同写法）。不调用 `GetNewX25519Cert`，不调用 `random.Base64Bytes`。
- **settings 形状**（VMess 顶层只有 clients 数组，**没有** decryption / fallbacks / encryption / method / password）：

```json
{
  "clients": [{
    "id": "<uuid.New().String()>",
    "security": "auto",
    "email": "<email>", "enable": true, "subId": "<subId>",
    "reset": 0, "limitIp": 0, "totalGB": 0, "expiryTime": 0,
    "tgId": 0, "comment": ""
  }]
}
```

- **streamSettings 形状**（与 ss2022 同形，security=none）：

```json
{ "network": "tcp", "security": "none", "tcpSettings": { "header": { "type": "none" } } }
```

- sniffing 复用 `oneClickSniffing()`。
- **坑**：
  - ❶ 不要输出 `alterId` —— 面板走 VMessAEAD，golden 里根本没有 alterId，加了反而是 legacy。
  - ❷ client 的 `security:"auto"` 存库没问题，但 `GenXrayInboundConfig→StripVmessClientSecurity` 会在下发 xray 前删掉它（设计如此，无害，正是手建 inbound 的行为）。
  - ❸ VMess-over-plain-TCP 无前向保密、易被 DPI 指纹识别 —— 能用但不如 reality，UI label 应注明「简单/调试用」。

## 预设 2：VLESS + TCP（security:"none"，无 REALITY 无 TLS，最简，免证书）

- **proto 常量**：`model.VLESS`
- **needs_cert**：false
- **keygen**：none —— 只需 `uuid.New().String()`。reality 版去掉 keygen 后的最简形态，无 privateKey/publicKey/shortIds。
- **settings 形状**（与 reality 版几乎相同，**唯一关键改动：flow 必须为空**）：

```json
{
  "clients": [{
    "id": "<uuid.New().String()>",
    "flow": "",
    "email": "<email>", "enable": true, "subId": "<subId>",
    "reset": 0, "limitIp": 0, "totalGB": 0, "expiryTime": 0,
    "tgId": 0, "comment": ""
  }],
  "decryption": "none",
  "fallbacks": []
}
```

- **streamSettings 形状**（删掉 realitySettings，security 从 reality 改 none）：

```json
{ "network": "tcp", "security": "none", "tcpSettings": { "header": { "type": "none" } } }
```

- sniffing 复用 `oneClickSniffing()`。
- **坑**：
  - ❶【最大坑】`flow` 必须是 `""`，**绝不能填 xtls-rprx-vision** —— vision flow 依赖 TLS/REALITY 传输层，security:"none" 下 xray 直接报错拒绝启动（`failed to use xtls flow without tls/reality`）。
  - ❷ `encryption` 字段：golden 带 `"encryption":"none"`，但 reality builder 本身不输出它，且 `GenXrayInboundConfig→StripVlessInboundEncryption` 会在下发前删掉 —— 照 reality builder 一样**省略 encryption** 即可（下发前反正会被 strip，省掉更干净）。
  - ❸ VLESS+TCP+none 基本是明文帧、极易被 DPI 阻断，只适合本地/测试/可信内网做基线或排障；生产免证书首选仍是 reality。UI label 标「最简/调试」。

## 新增一个一键预设需要改的位置（全部集中在 oneclick.go + 一个前端文件，model.go 无需改）

**后端 `internal/web/service/oneclick.go`：**
1. 新增 builder：仿 `ss2022InboundParts(email, subId)`（无 keygen 型）写 `vmessTcpInboundParts` / `vlessTcpInboundParts`，签名 `(email, subId string) (settings, stream, sniffing map[string]any)`，sniffing 一律 `oneClickSniffing()`。（需要密钥的协议才仿 `realityInboundParts` 带 priv/pub 参数。）
2. `switch req.Protocol`（约 226 行）加 `case "vmess"` / `case "vlessTcp"`，各设 `proto = model.VMESS / model.VLESS` 并调对应 builder。**因无 keygen，分支就是 ss2022 那样两行**，不要抄 reality 里 `i==0` 的 keygen-fast-fail 段。
3. **预设归一化（约 163-166 行）**：现在 `if req.Protocol != "ss2022" { req.Protocol = "reality" }` 会把任何未知值静默吞成 reality。**必须改成白名单式**（判断 known presets：reality/ss2022/vmess/vlessTcp），否则新预设被静默改回 reality。⚠️ 这是加协议时最容易漏的一步。
4. RemarkPrefix 默认名（约 167-173 行）：为新预设补默认前缀，vmess→"vmess"、vlessTcp→"vless"。
5. （文档）更新顶部 `OneClickRealityRequest` doc 注释里列出的预设清单。

**前端 `frontend/src/pages/inbounds/InboundsPage.tsx`：**
6. Select options（约 839-842 行）加：`{ value: 'vmess', label: 'VMess + TCP（免域名/证书，简单）' }`、`{ value: 'vlessTcp', label: 'VLESS + TCP（免证书，最简/调试）' }`。
7. 额外输入项 gate：dest 输入只在 `oneClickProto === 'reality'` 显示（约 854 行）。VMess/VLESS-none 无需额外字段。
8. 更新 remarkPrefix 占位符文案（约 852 行，现写「reality / ss」）把新默认名列上。

**无需改动：**
- ❶ 接口路径仍 `/panel/api/inbounds/oneclick/reality`（历史名，真正选预设的是 body 里 protocol 字段），controller/route 不动。
- ❷ model.go 不动 —— `Inbound.Protocol` 的 validate `oneof=vmess vless trojan shadowsocks …` 已允许 vmess/vless。
- ❸ 持久化仍走 `s.AddInbound(inbound)`，tag 生成、client_records 落库、xray 热加载全自动，与 reality/ss2022 完全同路径。

**验证建议**：新预设生成后对照 golden 的 `vmess-tcp-tls.json` / `vless-ws-tls.json` 比对 settings/streamSettings 形状（security 换 none、VLESS flow 换空），并实机跑一次 xray 重启确认 core 真加载（VLESS flow 非空是最常见启动失败原因）。
